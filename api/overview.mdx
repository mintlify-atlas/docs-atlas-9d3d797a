The Mikoto API is a RESTful HTTP API that powers the Mikoto messaging platform. All requests are made to the base URL and use JSON for both request and response bodies. A persistent WebSocket connection provides real-time events for messages, channels, spaces, and presence.

## Base URL

```
https://api.mikoto.io
```

Self-hosted deployments use the URL configured in your server's `API_URL` environment variable. The API server runs on port `3511` by default.

## Request format

All write operations (`POST`, `PATCH`, `PUT`) expect a JSON body. You must include the `Content-Type: application/json` header on every request that has a body.

```bash
curl -X POST https://api.mikoto.io/spaces \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <access_token>" \
  -d '{"name": "My Space"}'
```

## Authentication

The API uses JWT bearer tokens. Include the token in the `Authorization` header on every authenticated request.

```
Authorization: Bearer <access_token>
```

Access tokens are short-lived. Use the refresh token returned at login to obtain a new access token when the current one expires. See the [Authentication](/api/authentication) page for details.

<Warning>
Never expose your access token or refresh token in client-side code, public repositories, or URLs. Tokens in URLs are logged by proxies and browsers.
</Warning>

## Rate limiting

Authentication endpoints (`/account/register`, `/account/login`) are rate-limited per IP address to protect against brute-force attacks. If you exceed the limit you will receive a `429 Too Many Requests` response. Back off and retry after a short delay.

## Error responses

All errors return a JSON object with a machine-readable `code` field and a human-readable `message`.

```json
{
  "code": "NotFound",
  "message": "The requested resource was not found."
}
```

Common HTTP status codes:

| Status | Meaning |
| ------ | ------- |
| `200` | Success |
| `400` | Bad request — check your request body |
| `401` | Unauthorized — token missing or invalid |
| `403` | Forbidden — you lack the required permission |
| `404` | Not found |
| `409` | Conflict — e.g. resource already exists |
| `429` | Rate limited |
| `500` | Internal server error |
