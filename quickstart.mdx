---
title: "Get started with Mikoto in minutes"
description: "Create your account, set up your first space, add channels, and invite members to your Mikoto community — no installation required."
---

Mikoto is available as a hosted service at [platform.mikoto.io](https://platform.mikoto.io). You can have a community space running with channels and members in a few minutes. If you want to run Mikoto on your own infrastructure instead, see [Self-Hosting](/self-hosting).

<Steps>
  <Step title="Create an account">
    Go to [platform.mikoto.io](https://platform.mikoto.io) and register for an account. You'll need to provide an email address and password. After registration, verify your email address using the link sent to your inbox.
  </Step>

  <Step title="Create your first space">
    A **space** is the top-level container for your community — similar to a server or guild. After logging in:

    1. Click the **+** icon in the left sidebar.
    2. Choose **Create a Space**.
    3. Enter a name for your space.
    4. Click **Create**.

    Your new space opens immediately and you are its owner.
  </Step>

  <Step title="Add channels">
    Channels live inside a space and determine what kind of conversation happens there. Mikoto supports three channel types:

    | Type | Description |
    | --- | --- |
    | **Text** | Thread-based messaging with attachments |
    | **Voice** | Real-time audio/video powered by LiveKit |
    | **Document** | Live collaborative wiki editing via Y.js |

    To add a channel:

    1. Click **+** next to the channel list in your space.
    2. Select the channel type.
    3. Enter a channel name and click **Create**.

    <Tip>Start with one text channel (e.g., `#general`) and one document channel for community notes. You can add more channels at any time.</Tip>
  </Step>

  <Step title="Invite members">
    Share your space with others using an invite link:

    1. Open **Space Settings** from the sidebar.
    2. Navigate to the **Invites** tab.
    3. Click **Create Invite** to generate a shareable link.

    Anyone with the link can join your space. You can revoke invite links from the same settings panel.
  </Step>

  <Step title="Create a bot using mikoto.js (optional)">
    If you want to automate actions or build integrations, Mikoto provides a first-class JavaScript SDK called `mikoto.js`. The example below connects a bot and sends a message to a channel.

    First, create a bot account:

    1. Go to [platform.mikoto.io](https://platform.mikoto.io) and open **Developer Settings**.
    2. Click **New Bot** and enter a name.
    3. Copy the **Bot ID** and **Token** shown after creation.
    4. Install the bot into your space from the **Bots** section of Space Settings.

    Then install the package and connect:

    ```bash
    npm install @mikoto-io/mikoto.js
    ```

    ```typescript bot.ts
    import { BotAuthClient, MikotoClient } from '@mikoto-io/mikoto.js';

    const auth = new BotAuthClient({
      url: 'https://api.mikoto.io',
      botId: 'YOUR_BOT_ID',
      token: 'YOUR_BOT_TOKEN',
    });

    const client = new MikotoClient({
      auth,
      url: 'https://api.mikoto.io',
    });

    // Listen for new messages and respond to commands
    client.ws.on('messages.onCreate', async (msg) => {
      if (msg.content !== '!ping') return;

      const channel = client.channels.cache.get(msg.channelId);
      if (!channel) return;

      // Send a reply
      await channel.sendMessage('pong!');
    });
    ```

    <Note>
      `BotAuthClient` authenticates using a bot ID and secret token instead of user credentials. It overrides the `refresh()` method inherited from `AuthClient` to call the `bots.login` endpoint automatically. You do not need to manage tokens manually.
    </Note>

    The `MikotoClient` exposes two primary managers for navigating your community:

    - `client.spaces` — a `SpaceManager` that lists, joins, and caches `MikotoSpace` objects
    - `client.channels` — a `ChannelManager` that caches `MikotoChannel` objects across all spaces

    Call `channel.sendMessage(content)` to post a message, or `channel.listMessages(limit, cursor)` to read message history.
  </Step>
</Steps>
